WeRateDogs Data Wrangling and Analysis Project

Summary of Wrangling Section of Project

Mark MacArdle

11/05/2018

Gather Step

Files were imported or scraped into the three data frames highlighted in bold below.

  • Imported the twitter-archive-enhanced.csv file with pandas.read_csv() function into a dataframe called tweetArchive.
  • Used the requests library and it’s .get() function to programmatically download the image-predictions.tsv file. Then read that into dataframe called tweetBreedPredictions with .read_csv() again.
  • Got twitter api access tokens from https://apps.twitter.com/ and used these with the tweepy library to loop through the tweet ids from the twitter-archive-enhanced.csv to pull in extra information about the tweets in JSONs.
  • Wrote these JSONs into a text file called tweetJSONs.txt with the json library and the .dump() function
  • Then reimported this into a dataframe called twitterAPIdf with the pandas function .read_json().

Assessing Step

Used the .info(), .describe() and .head() functions to get an idea of the contents of the dataframes. Also used .value_counts() and .sort_values() to see the common unique values in a column.

tweetArchive Dataframe

  • Assessed visually in Google Sheets as well as programmatically
  • There were 4 columns describing the age phase of a dog ('doggo', 'floofer', 'pupper', 'puppo'). The values in these columns were either the column heading repeated or ‘None’. I made a function called countNotNones to find that of the 2356 rows in the dataframe 366 had one age phase identified and 14 had two.

twitterAPIdf Dataframe

  • Plotted favorite_count and retweet_count columns on histograms to see if the amount of zero values in those made sense. Retweet_count looked natural but favorite_count had a huge spike at 0.
  • Checked visually to find they seemed to be retweets.
  • Confirmed programmatically by finding they all had a retweeted_status_id value.

tweetBreedPredictions Dataframe

  • Confirmed that tweet ids weren’t repeated in the dataset by using .value_counts().max() to see that each tweet id only occurred once.

Cleaning Step

First made copies of the dataframes with the suffix “_clean” . Then cleaned 10 issues and tidied 3. Used .info() and .value_counts() a lot to test that fixes had worked.

Cleaned Issues

tweetArchive Dataframe

  1. Removed rows that were retweets and removed columns to do with retweets.
  2. Converted the timestamp column to datetime datatype
  3. Converted the tweet_id column to string (called object in pandas) datatype
  4. (My favourite!) Fixed mistaken rating numerators. In the given file if a rating had a decimal such as 9.75/10 it was mistakenly taken as 75/10. I used a regex expression and the python re library to loop all the text from tweets and extract the full numerator properly.
  • It was the first time I’d used a regex in practice and I very chuffed with getting it to work!
  1. Corrected other erroneous numerator / denominator values that had been identified by looking at outlying values for each (eg 1/2 or 50/50)
  2. Removed rows that were replies and then removed columns to do with that.

twitterAPI Dataframe

  1. Converted id column to string data type and renamed it to tweet_id to match other dataframes.
  2. Dropped all rows with a favorite_count of 0 as they were retweets.

tweetBreedPredictions Dataframe

  1. Converted tweet_id column to string data type
  2. Renamed 9 columns with more descriptive names, eg changed ‘p1’ to ‘most_likely_breed’
  3. (Actually done in Tidiness Issue #3 step) Non-dog predictions for most_likely_breed were replaced with NaNs.

Tidied Issues

  1. In the tweetArchive the doggo, floofer, pupper and puppo columns were converted into one categorical column called dog_age_stage. ‘None’ values were replaced with NaNs.
  2. The favorite_count and retweet_count columns in twitterAPIdf were merged into tweetArchive.
  3. The most_likely_breed and most_likely_breed_is_dog columns from in tweetBreedPredictions were merged into twitterArchive.
