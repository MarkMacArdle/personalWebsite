Loss Aversion in Penalty Shoot-Outs (Python, Pandas, Beautiful Soup)


Link to Github

Link to blog post

I wanted to see if loss aversion is playing an effect in world cup penalties shoot-outs. I explain what loss aversion is fully and what I did to test if it was present in the blog post linked above. The short summary is that I was testing to see if the outcome of a kick was influenced by the preceding kick. There was a measured difference but it wasn’t significant.

I using the Python libraries Beautiful Soup and Pandas to scrape the data from Wikipedia and then wrangled it. For the hypothesis test I used a Chi-Squared test as this was categorical data.

UK Vs China Air Pollution (Tableau)


Link to project

I used data from the World Health Organisation to compare if UK air pollution was still better than China's. It may seem obvious that the UK's would be better but I had been reading in the news that the UK government has lost three court cases over how bad it's air pollution plans are. Meanwhile China has been receiving praise for the improvements it's been making so I wanted to check if the UK was still ahead. Unfortunately data was only available between 2013 to '16 but it was still possible to create an interesting story.

This was the project for the data visualisation section of the Data Analyst Nanodegree on Udacity. They had some simpler data to do the project on but I was curious about this issue so opted to source my own data.

Exploration of Red Wine Data Set (R)


Link to project (the nicest plots are at the end!)

Link to GitHub

Exploratory data analysis (EDA) of red wine data set with R for the EDA section of the Data Analyst Nanodegree on Udacity. This was my first time using R and the course that built up to this project is available for free here. It goes through how to use R and R studio as well as what EDA is. The only difference between the free course and Nanodegree version is that you won't be assigned this project at the end. However if you do want to do the project the source data is available on my GitHub.

It follows the structure of univariate, bivariate and multivariate analysis with a summary of findings or points or interest at the end of each section. There's then a final plots and conclusion section with 3 of the plots from the analysis polished up to look more presentable. Can you guess what the variable with the highest correlation to the wine's rating out of 10 was? ...It was, of course, alcohol content! Higher alcohol percentages had a positive relationship with higher ratings.

I was really surprised how much I liked R. Before this I'd thought it was just something used by academics who hadn't got with the times and learned Python yet. I still think Python is better for working with data but R and its ggplot2 library are actually great for creating graphs. I thought it was much easier to build up complex, multi-layered plots than it would be in Python and its graphing library Matplotlib.

Country Mover - aka Africa is Huge (Google Maps API, Javascript)

Link to page

Courses

Below, again with the most recent first, is a list of the courses I've done so far in my own time and my experience with them. I take a lot of notes so I’ve included links to these in case they may be of use to your learning too. Be warned though that I wrote them for myself and wasn’t thinking of others using them at the time!

Data Analyst Nanodegree

Course link

My notes on: R, data wrangling with Python, data visualisation with Tableau

My certificate for completing the course.

I did term 2 of this course on Udacity which had three sections:

  1. Exploratory data analysis with R (course available free here)
  2. Data wrangling with Python and the Pandas library
  3. Data visualisation with Tableau (course available free here)
It was the first course I did on Udacity and I was very impressed with it. It's delivered mainly through numerous short video lectures which are typically only 2-3 minutes long and normally have an exercise to do during or at the end of each video. It was a lot more expensive than courses I've done before at £579. A lot of the content is available free, as linked above. The advantage of the paid version is at the end of each section has a relatively large project to do that gets graded. As well as the benefit of doing a project yourself this also means you have something to show off your work with.

During this course I've found I needed to brush up on a lot of things so have also done additional smaller courses on using Git for version control (which I now use for the files of this website), Linux command line basics to help with using Git, intros to descriptive and inferential statistics (which I flicked through to refamiliarise myself on statistics terms), and an intro to the Python Pandas library which I'd used before but never fully understood.

Projects

The was a project at the end of each section. They're detailed better in the main Projects section of this site but in summary were explorative data analysis on a red wine data set, gathering and cleaning data from the WeRateDogs twitter profile and visualising the differences between UK and China's air pollution in big cities.


Time Taken

Not including some of the extra courses I did it took 106 hours in total. The first, second and third section took 45, 43.5, and 17.5 hours respectively. The projects took 17, 21.5 and 10.5 hours respectively. So the final section on data visualisation was much shorter than the other two.

SQL Bootcamp

Course link

My notes link

Course used SQL in PostGres with pgAdmin and was provided on Udemy. It started with basic queries of a database to select and view columns before moving on to more advanced with conditional queries and topics like joining or self-joining tables. It also covered setting up new databases and creating and modifying tables. I feel like it ended a little too soon as the creating tables section seemed to leave out how to use foreign keys in different tables.

Overall though it was an enjoyable and clear course. It was thought by the same tutor as the "Python Bootcamp" course I'd previously done, which I also enjoyed.

Projects

The assessments and challenges were on querying databases of a DVD rental store social club. Both had multiple tables that you sometimes had to join to properly query. In the final section you had to create your own database with teacher and student tables, define the data types and insert rows of information.

While the activities were frequent through the course they were all quite brief. This is a common criticism I have of Udemy courses as it's hard to know much you can do on your own without challenges that need significant amount of independent work.


Time Taken

This was likely the shortest course I've done. It had 8.5 hours of video but only took 15 hours for me to get through, which is largely due to the short activities.

Machine Learning

Course link

My notes link

This was the first course I’ve done on Coursera. It was created by Stanford University which is famous for computer science and for a huge amount of its students going on to work at the major tech companies. The lecturer is Andrew Ng who himself is a famous for advances his teams have made in machine learning. This course is a shortened version of what they teach on campus (which is available for free too). Also if you google or search on Quora for how to learn machine learning you’ll very likely see this recommended.

With all that credibility and popularity I had pretty high expectations. I did really enjoy this course and found it fascinating, but the thing that no one seemed to bother mentioning on Quora is that it’s mainly about the theory of machine learning, rather than applying it. This means enough mathematical notation to suffocate a tall elephant and any programming is done in Matlab (or the free equivalent Octave). There’s an optional brush up section at the start on linear algebra that I recommend because matrices and their multiplication feature heavily.

The course content is almost the opposite of the Data Science course I did previously on Udemy. That was all about how to apply it and showing the code you’d use in Python, this was about how the algorithms behind it all worked. If this was the first thing I tried to do in machine learning I probably would have run away, but after the other course this was actually really interesting to see how it all worked.

Projects

It has 8 projects that took me 4 to 5 hours to complete each week. They’re in Matlab and involve finishing functions to get a program they’ve written to work. While they were good for increasing my understanding of the material, it suffers a similar problem to the Data Science course. Now that I’ve finished it I’m not much better at applying machine learning independently, despite understanding it better. Like the Data Science course to make the most of it you’ll have to independently find some project to work on to apply your learning. I found Kaggle’s initial challenges like the Titanic data set great for this.


Time Taken

I’m not sure how many hours of video lectures there are but it took me 82 hours over 10 weeks overall to complete. Unlike the Udemy courses this is broken down to have a weekly structure with one assignment and a multiple choice test or two to complete each week. You’ll get reminders to complete the assignments on time. There’s actually no penalty for submitting late but I found the structure helpful anyway as it encouraged me to stay on track.

Complete Python Bootcamp

Course link

My notes link

I did this course on Udemy and it is my favourite so far. I feel like I’ve gained a really good (or at least vastly improved) understanding of Python and was able to apply it by creating a two player version of Snake (check it out on my GitHub ). It covered data structures and object orientated programming. The data structures content, like working with lists and strings, is exactly what you need to know for code challenges and HackerRank questions. I don’t really have anything negative to say about it.

Projects

There's three milestone projects as you work through the course. A text based tic-tac-toe game after you've learnt about logic and loops. A text based blackjack game after learning about functions and classes and then after some more work you're directed to choose your own project that will take about a week. He does give some ideas to help with choosing. I chose to do a two player version of Snake as I wanted to practice using classes more.


Time Taken

The course has around 12 hours of videos and it took me 84 hours over 9 weeks to complete. I’d say I spent half of that time or more on the projects.

Data Science and Machine Learning with Python

Course link

My notes link

I found this course really interesting and got through it pretty quickly. It goes through and demonstrates the most common machine learning techniques using Python in Enthought Canopy. The lecturer, Frank Kane, has interesting experience having worked for IMDB and is engaging to listen to. His explanations of how the different algorithms work are clear and concise and he doesn’t get bogged down in the maths behind it all. The format of the lectures is typically a few slides on how an algorithm works and then talking through an IPython notebook where he'll demo what each part of the code does.

Projects

Maybe his lack of teaching experience shows here as I felt the assignments were the weak part of this course. There are frequent "activities," but there are mainly change the variables in code he has written and see the effect on the result. This doesn't give you great experience as a writing the code the get to the stage where you apply the algorithm is actually a pretty big part of it. Possibly if I'd been better at Python at the time I wouldn't have considered it such an issue.

That doesn't mean the course was a waste, just that to make the most of it you’ll just need to find some projects that will let you practice independently applying your learning. I found Kaggle’s initial challenges like the Titanic data set great for this.


Time Taken

It’s a fairly short course with about about 9 hours of video content and with the lack of independent work to do means you can get though it quickly. It took me 41 hours over 9 weeks to complete. I did spend a good few of those hours trying to unsuccessfully create that recommender engine though!

The Complete Web Developer Course 2.0

Course link

My notes link

This was the first course I did and it’s also the only one I haven’t actually finished it (I’m at 75%!). It’s one of the most popular courses on Udemy and definitely covers a lot of material for full stack web development. It does what it says on the tin and covers html, CSS, JavaScript, JQuery, Bootstrap, and to what I felt was a less thorough extent, PHP and MySQL. There's also a Python bit that I haven't got to yet. It’s from Rob Percival who has some of the most popular courses on Udemy. It’s well structured with a lot of assignments to implement your learning by creating your own web pages.

One criticism I have of this course is that I feel a bit of detail on data structures of different languages is passed over. For example in the JavaScript section arrays are introduced as a way to store a list of values, but the fact there are other ways to store values or that they have different performance when being searched isn't.

Projects

Despite skipping over a some of the detail the projects you do are impressive. For example see some of web pages I made in the Projects section, like the JSbin clone. The focus on projects is definitely a highlight of the course and gives a real sense of accomplishment. Every section has a project at the end to create a web page with what you've learned. Also during the videos he continuously prompts you to pause and try what he's been demonstrating. You could do this anyway but his persistence helps stop lasyness getting the better of you.


Time Taken

When I was doing this course I hadn't started recording the amount of hours I was spending on programming so I don’t have an exact figure for what it took me. It's a long course with about 30 hours of video content. Relating that to the Python course I did, in which I think I spent a similar proportion of time on projects, and I’d say I’ve spent about 150 hours on it.

Course Providers Comparison: Udemy, Coursera and Udacity

So far the websites I've used for my courses are Udemy, Coursera and Udacity. I’ve found them all good but with some differences between them which I discuss below. This is just based on my own experiences with them so isn't definitive and your experience may vary!


Udemy

I feel the courses on Udemy were to most focused on getting something to show for your work. The courses are normally by professionals who have good industry experience but they may not have taught before. For example the data science with Python course I did had good content and was very focused on what’s asked for in job ads. But it didn’t have many projects you had to do independently so I struggled to apply what I’d learnt afterwards. That said there can be a lot of variation between courses and I thought the Python Bootcamp or Complete Web Developer courses I did were examples of well structured courses with good practical exercises.

The courses vary in length but the bulk seem to have about 20 hours of video lectures. These are normally broken down into 5 to 10 minute videos focusing on particular issues so you could hope through the courses to get to what you want.

Costs
They advertise courses for around £200 but have near-perpetual sales so I haven’t paid more than £15. They’ll always say the sale is just about to end, but really it’ll be back on sale in a few days so don’t fell rushed into purchases. A tip I've found is to download the Udemy app before you buy a course. On more than one occasion when on a computer the Udemy website will advertise a course to me for £15, but when I check the same course on the app it will be £10. Now I purchase my courses using the app and do them on my laptop.


Coursera

On Coursera the courses are from universities and delivered by lecturers so they have great teaching experience but are more academic. The machine learning course also had good content but also had programming assignments each week that took me around 4-5 hours to do and definitely increased my understanding of the material. The downside was the material was quite theoretical and those assignments were in Matlab, a language popular in academia but not industry. I think (like my memories of actual university) they can focus too much on the theory and not enough in applying it in the real world.

Costs
I've found Coursera's costs to be quite changeable so you'll have to check their website for the latest. When I first looked at a course in 2015 all the courses were free and they sold upgrades like a cert for your LinkedIn profile. When I did my second course in July 2017 (on machine learning, detailed in its own post) each University seemed to have a few entry level courses for free and then charge for the advanced material.


Udacity (My new favourite!)

I think Udacity is better than both Udemy and Coursera by having the best balance of a well structured course that covers the relevant theory but also covers applying it in the real world. Their videos also have the highest productiton quality I've seen so far and often pause midway to get you to do small exercises which helps make them engaging.

Costs and Course Types
Udacity are by far the most expensive but their mission is to "democratize education" so they have a lot of content in their catalogue which is free for anyone to view. I don't think all the latest videos are there but the most are. For instance of the 3 sections of the Data Analyst Nanodegree I did two of the video courses are available for free.

The second way is by paying for their "Nanodegree" courses, which cover a few courses within them (see my post on the Data Analyst one for more detail). I paid £579 for term 2 of the Data Analyst Nanodegree Term 2 and generally the Nanodegrees seem to be between £300 to £600. I've noticed the prices vary seasonally. There were discounts being offered last summer, then none at the end of the year and start of a new year. Now in May I see some courses getting discounted again.

For all this money the extras you get over the free courses is that you have assignments to do, which are actually graded by a human, and a deadline to finish them by as well as help with CVs and interview practice. I've only been interested in the assignments so far and found them really useful. They've taken about 20 hours to do and are structured to end up with something presentable at the end. Hence they are very good for having something tangible to show for you efforts or discuss in an interview.

The fact they're graded by a human wasn't that big a deal to me. The assignment requirements are normally quite specific so you can tell if you've met them before you submit. If they do send you back corrections though they are helpful in describing the issue and how to approach it better.

Are The Paid Courses Worth It?
For me, yes. You definitely could save the money just doing the free courses and find some data sets to do projects on. However I've found it very hard to know where to start when doing that. Also motivating yourself to do a project when there's no time pressure and getting that project to a high and presentable level when there's no requirements is hard! This is also my hobby and I really enjoy doing these courses which makes it a lot easier to be comfortable spending the money.
